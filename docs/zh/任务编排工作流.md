# 任务编排工作流（Task Orchestration Workflow）

本指南说明主控智能体（master agent / orchestrator）应该如何编排子智能体，使其正确地与 Mission Control 集成。

## 概览（Overview）

当 orchestrator 为某个任务创建一个子智能体时，**所有活动记录、交付物和会话信息都必须写回 Mission Control**，这样 UI 才能实时展示进度。

## 导入辅助库（Import the Helper）

```typescript
// 在 Node.js 环境中（orchestrator 所在的运行环境）
import * as orchestrator from '@/lib/orchestration';

// 如果无法使用 TypeScript 模块，也可以直接用 fetch 调后端 API
```

## 工作流步骤（Workflow Steps）

### 1. 创建子智能体时（When Spawning a Sub-Agent）

在**刚刚创建子智能体之后立刻**注册会话：

```typescript
await orchestrator.onSubAgentSpawned({
  taskId: 'task-abc123',                           // Mission Control 中的任务 ID
  sessionId: 'agent:main:subagent:xyz789',         // 子智能体的 OpenClaw 会话 ID
  agentName: 'fix-mission-control-integration',    // 便于辨认的名字
  description: 'Fix real-time updates and logging', // 可选说明
});
```

**这一调用会做什么：**

- 在 `task_activities` 中写入一条活动记录：  
  `"Sub-agent spawned: fix-mission-control-integration"`
- 在 `openclaw_sessions` 表中插入一条会话记录，`session_type = 'subagent'`  
- 通过 SSE 广播事件，让前端 UI 立刻更新  
- 侧边栏中的智能体计数从 0 → 1

### 2. 子智能体工作过程中（During Sub-Agent Work）

在子智能体的关键步骤上报活动：

```typescript
await orchestrator.logActivity({
  taskId: 'task-abc123',
  activityType: 'updated',
  message: 'Fixed SSE broadcast in dispatch endpoint',
  metadata: { file: 'src/app/api/tasks/[id]/dispatch/route.ts' }
});

await orchestrator.logActivity({
  taskId: 'task-abc123',
  activityType: 'file_created',
  message: 'Created orchestration helper',
  metadata: { file: 'src/lib/orchestration.ts' }
});
```

**活动类型（Activity Types）：**

- `spawned` —— 子智能体已启动  
- `updated` —— 一般性的进度更新  
- `completed` —— 子智能体完成任务  
- `file_created` —— 创建或修改了文件  
- `status_changed` —— 任务状态发生变更

### 3. 子智能体完成时（When Sub-Agent Completes）

在把任务状态变更为 `review` 之前，先记录完成情况及交付物：

```typescript
await orchestrator.onSubAgentCompleted({
  taskId: 'task-abc123',
  sessionId: 'agent:main:subagent:xyz789',
  agentName: 'fix-mission-control-integration',
  summary: 'All integration issues fixed and tested',
  deliverables: [
    {
      type: 'file',
      title: 'Updated dispatch route',
      path: 'src/app/api/tasks/[id]/dispatch/route.ts'
    },
    {
      type: 'file',
      title: 'Orchestration helper',
      path: 'src/lib/orchestration.ts'
    },
    {
      type: 'file',
      title: 'Fixed Header component',
      path: 'src/components/Header.tsx'
    }
  ]
});
```

**这一调用会做什么：**

- 写入“完成”类活动记录  
- 将 `openclaw_sessions` 中对应会话的 `status` 设为 `completed`，并写入 `ended_at` 时间戳  
- 把所有交付物写入 `task_deliverables` 表  
- 通过 SSE 广播事件，前端 UI 实时更新  
- 侧边栏的智能体计数从 1 降回 0

### 4. 审核与通过（Review & Approval）

在真正把任务从 `review` → `done` 之前，需要**先检查该任务是否已有交付物**：

```typescript
const hasDeliverables = await orchestrator.verifyTaskHasDeliverables('task-abc123');

if (!hasDeliverables) {
  console.log('⚠️ Task has no deliverables - cannot approve');
  console.log('📋 Ask sub-agent to provide deliverables or log them manually');
  return;
}

// 现在可以安全地批准任务
await fetch('http://localhost:4000/api/tasks/task-abc123', {
  method: 'PATCH',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    status: 'done',
    updated_by_agent_id: 'orchestrator-agent-id'  // orchestrator 在 agents 表中的 ID
  })
});
```

**后端的校验逻辑：**

- 如果任务从 `review` 变更到 `done` 时没有任何交付物，API 会返回 400 错误  
- 只有 master agent 才能批准任务  
- 这确保了最基本的交付质量

## 不使用辅助库时的直接 API 调用（Direct API Usage Without Helper）

如果你无法 import `@/lib/orchestration`，可以直接通过 HTTP API 操作：

```typescript
// 注册子智能体
await fetch('http://localhost:4000/api/tasks/TASK_ID/subagent', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    openclaw_session_id: 'agent:main:subagent:xyz',
    agent_name: 'my-subagent-name'
  })
});

// 记录活动
await fetch('http://localhost:4000/api/tasks/TASK_ID/activities', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    activity_type: 'updated',
    message: 'Did something important'
  })
});

// 记录交付物
await fetch('http://localhost:4000/api/tasks/TASK_ID/deliverables', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    deliverable_type: 'file',
    title: 'My deliverable',
    path: 'path/to/file.ts'
  })
});

// 结束会话
await fetch('http://localhost:4000/api/openclaw/sessions/SESSION_ID', {
  method: 'PATCH',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    status: 'completed',
    ended_at: new Date().toISOString()
  })
});
```

## 测试清单（Testing Checklist）

在实现完这一编排工作流后，确认：

- ✅ 任务状态变化可以在 UI 中实时看到，无需刷新  
- ✅ 当有子智能体工作时，智能体计数会显示为 1  
- ✅ 活动日志中会显示所有时间戳、操作信息  
- ✅ 交付物标签页里能看到所有文件 / 产物  
- ✅ 会话标签页能展示子智能体会话及其开始 / 结束时间  
- ✅ 顶部统计区域能准确显示 “X agents active, Y tasks in queue”  
- ✅ 没有交付物时无法把任务批准为 `done`

## 常见坑（Common Pitfalls）

1. **忘记调用 onSubAgentSpawned** → 侧边栏中的智能体计数始终为 0  
2. **不记录交付物** → 任务无法通过最终审批  
3. **sessionId 格式错误** → 后端找不到会话记录  
4. **不调用 onSubAgentCompleted** → 智能体计数不会减回去  
5. **跳过 verifyTaskHasDeliverables 就直接批准** → 后端返回 400 错误

## 完整示例（Example: Complete Workflow）

```typescript
// 1. 创建子智能体
const sessionId = await spawnSubAgent({
  label: 'fix-integration',
  task: taskDescription
});

// 2. 立刻在 Mission Control 中注册子智能体会话
await orchestrator.onSubAgentSpawned({
  taskId: task.id,
  sessionId: sessionId,
  agentName: 'fix-integration',
  description: 'Fix Mission Control integration'
});

// 3. 过程中可以按需记录进度（省略具体代码）
// ...

// 4. 完成后一次性记录总结和交付物
await orchestrator.onSubAgentCompleted({
  taskId: task.id,
  sessionId: sessionId,
  agentName: 'fix-integration',
  summary: 'Fixed all integration issues',
  deliverables: [
    { type: 'file', title: 'Fixed route', path: 'src/api/...' }
  ]
});

// 5. 把任务状态改为 review
await updateTaskStatus(task.id, 'review');

// 6. 审核前最后检查是否有交付物
const hasDeliverables = await orchestrator.verifyTaskHasDeliverables(task.id);
if (hasDeliverables) {
  await updateTaskStatus(task.id, 'done', { updated_by_agent_id: orchestratorId });
} else {
  console.log('⚠️ Cannot approve - no deliverables');
}
```

---

**记住：** 每一个子智能体的行为都应该在 Mission Control 中“可见”。**没有被记录到系统里的事情，就等于没有发生。**

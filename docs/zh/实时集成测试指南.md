# 实时集成测试指南（Real-Time Integration Testing Guide）

## 快速开始（Quick Start）

```bash
cd ~/Documents/Shared/mission-control
npm install
npm run dev
```

在浏览器中打开 `http://localhost:4000`（生产服务或本地同一端口）。

## 测试场景（Test Scenarios）

### 1. SSE 连接测试（SSE Connection Test）

**需要验证：**

- 打开浏览器 DevTools → `Network` 面板 → 过滤关键字为 `"stream"`  
- 应该能看到到 `/api/events/stream` 的连接，状态为 `(pending)` 或 `200`  
- 连接应保持开启（不会立即关闭）  
- 控制台应打印：`[SSE] Connected`

**预期行为：**

- 连接在 1–2 秒内建立  
- 每隔约 30 秒会收到一次 keep-alive ping  
- 连接断开时会自动重连

---

### 2. 实时任务更新（Real-Time Task Updates）

**测试步骤：**

1. 在两个浏览器窗口中并排打开 Mission Control  
2. 在窗口 1 中点击 “+ New Task” 创建新任务  
3. 在窗口 2 中，任务应在 `INBOX` 列中自动出现（**无需刷新**）  
4. 在窗口 1 中，将任务拖动到 `ASSIGNED` 列  
5. 在窗口 2 中，任务应同步移动到 `ASSIGNED`，同样无需刷新

**预期行为：**

- 所有已连接客户端的任务卡片会实时新增 / 移动  
- 不需要刷新页面  
- 变更应在 1 秒内生效

---

### 3. 活动日志测试（Activity Log Test）

**测试步骤：**

1. 通过 API 或 UI 创建一个任务  
2. 打开该任务的详情弹窗（Task Modal）  
3. 点击 “Activity” 标签页  
4. 通过 API 发送 POST 请求记录活动：

```bash
curl -X POST http://localhost:4000/api/tasks/TASK_ID/activities \
  -H "Content-Type: application/json" \
  -d '{
    "activity_type": "updated",
    "message": "Task triaged and assigned to Developer agent",
    "agent_id": "AGENT_ID"
  }'
```

5. 活动应立即出现在活动列表中

**预期行为：**

- 活动按时间倒序显示（最新在最上方）  
- 每条活动展示：图标、智能体信息、消息内容、时间戳  
- 显示相对时间，如 “2 mins ago”、“1 hour ago”

---

### 4. 交付物测试（Deliverables Test）

**测试步骤：**

1. 打开某个任务的详情弹窗  
2. 点击 “Deliverables” 标签页  
3. 通过 API 发送 POST 请求添加交付物：

```bash
curl -X POST http://localhost:4000/api/tasks/TASK_ID/deliverables \
  -H "Content-Type: application/json" \
  -d '{
    "deliverable_type": "file",
    "title": "Implementation Report",
    "path": "~/Documents/report.md",
    "description": "Detailed implementation report"
  }'
```

4. 交付物应立即出现在列表中

**预期行为：**

- 每条交付物展示：图标、标题、描述、路径  
- 文件路径使用等宽字体（monospace）显示  
- 对于 URL 类型的交付物，有 “Open” 按钮并在新标签页打开

---

### 5. 子智能体跟踪测试（Sub-Agent Tracking Test）

**测试步骤：**

1. 打开某个任务的详情弹窗  
2. 点击 “Sessions” 标签页  
3. 通过 API 注册一个子智能体：

```bash
curl -X POST http://localhost:4000/api/tasks/TASK_ID/subagent \
  -H "Content-Type: application/json" \
  -d '{
    "openclaw_session_id": "agent:main:subagent:test-123",
    "agent_name": "Test Sub-Agent"
  }'
```

4. 列表中应出现新的子智能体会话  
5. 查看侧边栏，“Active Sub-Agents” 计数应增加

**预期行为：**

- Sessions 列表展示智能体头像、会话 ID、状态、持续时长等信息  
- 状态为 `active` 的子智能体显示绿色闪烁点  
- 侧边栏计数在最多 10 秒内更新（按轮询间隔）

---

### 6. 任务弹窗标签页测试（Task Modal Tabs Test）

**测试步骤：**

1. 打开任意任务  
2. 确认存在以下标签页：`Overview`、`Activity`、`Deliverables`、`Sessions`  
3. 依次点击每个标签，确认内容加载正常  
4. 确认 “Save / Delete” 按钮只出现在 `Overview` 标签内

**预期行为：**

- 切换标签不会关闭弹窗  
- 各标签内容相互独立加载，不互相阻塞  
- `Overview` 显示可编辑表单  
- 其他标签为只读数据视图

---

### 7. 多客户端 SSE 测试（Multi-Client SSE Test）

**测试步骤：**

1. 使用三种不同浏览器（Chrome / Firefox / Safari）打开 Mission Control  
2. 在浏览器 1 中创建或更新任务  
3. 确认所有浏览器几乎同时收到更新

**预期行为：**

- 所有客户端都能接收到 SSE 事件  
- UI 在各浏览器中实时更新  
- 不出现重复事件  
- 各浏览器控制台中能看到事件接收日志

---

## 集成测试：完整工作流（Integration Test: Full Workflow）

**场景：智能体编排（Agent orchestration flow）**

1. **orchestrator 创建任务：**

```bash
curl -X POST http://localhost:4000/api/tasks \
  -H "Content-Type: application/json" \
  -d '{
    "title": "Build authentication system",
    "description": "Implement JWT-based auth",
    "status": "inbox",
    "priority": "high"
  }'
```

2. **orchestrator 进行分拣并分配：**

```bash
TASK_ID="..." # 来自步骤 1 的返回

# 记录分拣活动
curl -X POST http://localhost:4000/api/tasks/$TASK_ID/activities \
  -H "Content-Type: application/json" \
  -d '{
    "activity_type": "updated",
    "message": "Task triaged and assigned to Developer agent"
  }'

# 更新任务状态为 assigned
curl -X PATCH http://localhost:4000/api/tasks/$TASK_ID \
  -H "Content-Type: application/json" \
  -d '{"status": "assigned"}'
```

3. **子智能体启动：**

```bash
# 注册子智能体
curl -X POST http://localhost:4000/api/tasks/$TASK_ID/subagent \
  -H "Content-Type: application/json" \
  -d '{
    "openclaw_session_id": "agent:main:subagent:dev-auth",
    "agent_name": "Developer Sub-Agent"
  }'

# 记录“子智能体启动”活动
curl -X POST http://localhost:4000/api/tasks/$TASK_ID/activities \
  -H "Content-Type: application/json" \
  -d '{
    "activity_type": "spawned",
    "message": "Spawned sub-agent for task execution"
  }'
```

4. **子智能体创建交付物：**

```bash
# 添加文件交付物
curl -X POST http://localhost:4000/api/tasks/$TASK_ID/deliverables \
  -H "Content-Type: application/json" \
  -d '{
    "deliverable_type": "file",
    "title": "auth.ts",
    "path": "~/project/src/auth.ts",
    "description": "JWT authentication implementation"
  }'

# 记录文件创建活动
curl -X POST http://localhost:4000/api/tasks/$TASK_ID/activities \
  -H "Content-Type: application/json" \
  -d '{
    "activity_type": "file_created",
    "message": "Created auth.ts with JWT implementation"
  }'
```

5. **子智能体完成：**

```bash
# 记录完成活动
curl -X POST http://localhost:4000/api/tasks/$TASK_ID/activities \
  -H "Content-Type: application/json" \
  -d '{
    "activity_type": "completed",
    "message": "Sub-agent completed task in 45 seconds"
  }'

# 将任务移动到 review
curl -X PATCH http://localhost:4000/api/tasks/$TASK_ID \
  -H "Content-Type: application/json" \
  -d '{"status": "review"}'
```

6. **在 UI 中验证：**

- 任务会在看板列之间实时移动  
- Activity 标签展示完整的时间线  
- Deliverables 标签展示 `auth.ts`  
- Sessions 标签展示子智能体（状态应该已完成）  
- 侧边栏中“Active Sub-Agents” 数字在子智能体结束时减少

## 故障排查（Troubleshooting）

### SSE 无法连接（SSE Not Connecting）

1. 检查浏览器控制台报错  
2. 验证 `/api/events/stream` 返回的 `Content-Type` 是否为 `text/event-stream`  
3. 检查是否有 CORS 相关问题  
4. 确认反向代理 / nginx 没有对 SSE 响应进行缓冲

---

### 事件未广播（Events Not Broadcasting）

1. 查看服务器日志中是否有 `broadcast()` 调用  
2. 确认在 DB 写入操作之后调用了 `broadcast()`  
3. 检查当前 SSE 客户端数量（浏览器应打印连接日志）  
4. 验证事件 payload 结构是否符合定义的 `SSEEvent` 类型

---

### UI 不更新（UI Not Updating）

1. 确认 SSE 连接处于活跃状态（Network 面板查看）  
2. 检查浏览器控制台是否能收到事件日志  
3. 确认 Zustand store 中 `updateTask` / `addTask` 等方法有被调用  
4. 确认组件已正确订阅 store 的状态变化

---

### 数据库错误（Database Errors）

1. 删除 `mission-control.db` 并重启应用（会重新创建 schema）  
2. 确认已开启外键约束  
3. 确认 SQLite 版本支持 JSON 和相关索引  
4. 检查数据库文件的读写权限

---

## API 接口参考（API Endpoint Reference）

### SSE 流（SSE Stream）

- `GET /api/events/stream` —— 建立 SSE 连接

### 活动（Activities）

- `GET /api/tasks/[id]/activities` —— 列出任务的所有活动  
- `POST /api/tasks/[id]/activities` —— 记录一条新的活动

### 交付物（Deliverables）

- `GET /api/tasks/[id]/deliverables` —— 列出所有交付物  
- `POST /api/tasks/[id]/deliverables` —— 新增交付物

### 子智能体（Sub-Agents）

- `GET /api/tasks/[id]/subagent` —— 列出子智能体会话  
- `POST /api/tasks/[id]/subagent` —— 注册子智能体会话

### OpenClaw 会话（OpenClaw Sessions）

- `GET /api/openclaw/sessions?session_type=subagent&status=active`  
  —— 统计当前所有状态为 `active` 的子智能体数量

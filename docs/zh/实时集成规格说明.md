# 实时集成规格说明（Real-Time Integration Specification）

## 目标（Goal）

为 Mission Control 的任务编排提供**完全透明**且**实时更新**的能力。

## 需求概览（Requirements）

### 1. 数据库结构扩展（Database Schema Extensions）

#### `task_activities` 表

```sql
CREATE TABLE task_activities (
  id TEXT PRIMARY KEY,
  task_id TEXT REFERENCES tasks(id) ON DELETE CASCADE,
  agent_id TEXT REFERENCES agents(id),
  activity_type TEXT NOT NULL, -- 'spawned', 'updated', 'completed', 'file_created', 'status_changed'
  message TEXT NOT NULL,
  metadata TEXT, -- JSON，附加上下文信息
  created_at TEXT DEFAULT (datetime('now'))
);

CREATE INDEX idx_activities_task ON task_activities(task_id, created_at DESC);
```

#### `task_deliverables` 表

```sql
CREATE TABLE task_deliverables (
  id TEXT PRIMARY KEY,
  task_id TEXT REFERENCES tasks(id) ON DELETE CASCADE,
  deliverable_type TEXT NOT NULL, -- 'file', 'url', 'artifact'
  title TEXT NOT NULL,
  path TEXT, -- 文件路径或 URL
  description TEXT,
  created_at TEXT DEFAULT (datetime('now'))
);

CREATE INDEX idx_deliverables_task ON task_deliverables(task_id);
```

#### `openclaw_sessions` 表增强（Enhance existing）

新增字段：

```sql
ALTER TABLE openclaw_sessions ADD COLUMN session_type TEXT DEFAULT 'persistent'; -- 'persistent' 或 'subagent'
ALTER TABLE openclaw_sessions ADD COLUMN task_id TEXT REFERENCES tasks(id);
ALTER TABLE openclaw_sessions ADD COLUMN ended_at TEXT;
```

### 2. WebSocket / SSE 服务（WebSocket Server）

> 设计上使用 WebSocket 作为抽象事件源，但实现推荐用 SSE。

#### 实现要点（Implementation）

- 使用 Next.js API 路由进行升级（或改用 SSE）  
- 端点：`/api/ws`（WebSocket）或 `/api/events/stream`（SSE）  
- 将事件广播给所有已连接客户端

#### 广播事件类型（Events to broadcast）

```typescript
type WSEvent =
  | { type: 'task_updated', payload: Task }
  | { type: 'task_created', payload: Task }
  | { type: 'activity_logged', payload: TaskActivity }
  | { type: 'deliverable_added', payload: TaskDeliverable }
  | { type: 'agent_spawned', payload: { taskId: string, sessionId: string, agentName: string } }
  | { type: 'agent_completed', payload: { taskId: string, sessionId: string, summary: string } };
```

### 3. 后端 API 接口（Backend API Endpoints）

#### `POST /api/tasks/[id]/activities`

记录任务活动。

```typescript
{
  activity_type: 'spawned' | 'updated' | 'completed' | 'file_created' | 'status_changed',
  message: string,
  agent_id?: string,
  metadata?: object
}
```

#### `GET /api/tasks/[id]/activities`

获取某个任务的所有活动（按 `created_at DESC` 排序）。

---

#### `POST /api/tasks/[id]/deliverables`

为任务添加交付物。

```typescript
{
  deliverable_type: 'file' | 'url' | 'artifact',
  title: string,
  path: string,
  description?: string
}
```

#### `GET /api/tasks/[id]/deliverables`

获取任务的全部交付物。

---

#### `POST /api/tasks/[id]/subagent`

为任务注册一个子智能体会话。

```typescript
{
  openclaw_session_id: string,
  agent_name: string
}
```

- 当任务变化时自动更新看板中的各列

### 5. 编排集成（Orchestration Integration）

orchestrator 在编排任务时的典型流程：

```typescript
// 1. 从 INBOX 中取出一个任务
const task = await fetch('http://localhost:4000/api/tasks?status=inbox').then(r => r.json());

// 2. 记录“分拣”活动
await fetch(`http://localhost:4000/api/tasks/${task.id}/activities`, {
  method: 'POST',
  body: JSON.stringify({
    activity_type: 'updated',
    message: 'Task triaged and assigned to Developer agent',
    agent_id: orchestratorAgentId
  })
});

// 3. 更新任务状态为 assigned
await fetch(`http://localhost:4000/api/tasks/${task.id}`, {
  method: 'PATCH',
  body: JSON.stringify({ status: 'assigned' })
});

// 4. 启动子智能体
const { childSessionKey } = await spawnSubAgent(task);

// 5. 在 Mission Control 中注册子智能体
await fetch(`http://localhost:4000/api/tasks/${task.id}/subagent`, {
  method: 'POST',
  body: JSON.stringify({
    openclaw_session_id: childSessionKey,
    agent_name: 'Developer Sub-Agent'
  })
});

// 6. 记录“子智能体启动”活动
await fetch(`http://localhost:4000/api/tasks/${task.id}/activities`, {
  method: 'POST',
  body: JSON.stringify({
    activity_type: 'spawned',
    message: 'Spawned sub-agent for task execution',
    metadata: { session_id: childSessionKey }
  })
});

// 7. 子智能体完成并生成文件后，记录交付物
await fetch(`http://localhost:4000/api/tasks/${task.id}/deliverables`, {
  method: 'POST',
  body: JSON.stringify({
    deliverable_type: 'file',
    title: 'Test Page',
    path: '~/Documents/Shared/mission-control/test-page.html',
    description: 'HTML test page with styling and dynamic content'
  })
});

// 8. 记录“完成”活动
await fetch(`http://localhost:4000/api/tasks/${task.id}/activities`, {
  method: 'POST',
  body: JSON.stringify({
    activity_type: 'completed',
    message: 'Sub-agent completed task in 20 seconds'
  })
});

// 9. 将任务状态更新为 review
await fetch(`http://localhost:4000/api/tasks/${task.id}`, {
  method: 'PATCH',
  body: JSON.stringify({ status: 'review' })
});
```

## 实现说明（Implementation Notes）

### WebSocket vs SSE（Next.js 环境）

Next.js 在 API 路由中不直接支持 WebSocket，需要额外处理。可选方案：

1. 使用 `ws` 库并自建服务器  
2. 使用 **Server-Sent Events（SSE）**，实现更简单、单向推送即可  
3. 使用外部 WebSocket 服务器（成本更高）

**推荐：采用 SSE** —— 与 Next.js 兼容性更好，实现成本低。

### SSE 端点：`/api/events/stream`

```typescript
export async function GET(request: NextRequest) {
  const encoder = new TextEncoder();
  const stream = new ReadableStream({
    start(controller) {
      // 注册客户端
      clients.add(controller);

      // 周期性发送 keep-alive
      const interval = setInterval(() => {
        controller.enqueue(encoder.encode(': keep-alive\n\n'));
      }, 30000);

      // 连接关闭时清理
      request.signal.addEventListener('abort', () => {
        clearInterval(interval);
        clients.delete(controller);
      });
    }
  });

  return new Response(stream, {
    headers: {
      'Content-Type': 'text/event-stream',
      'Cache-Control': 'no-cache',
      'Connection': 'keep-alive'
    }
  });
}
```

## 成功标准（Success Criteria）

1. 用户在 INBOX 中创建任务  
2. 在 60 秒内，能在界面上看到任务自动移动到 ASSIGNED（无需刷新）  
3. 侧边栏显示 “Active Sub-Agents: 1”  
4. 打开任务详情，在 Activity Log 中能看到类似以下条目：
   - `Task triaged and assigned to Developer agent`  
   - `Spawned sub-agent for task execution`  
   - `Created file: test-page.html`  
   - `Sub-agent completed task in 20 seconds`
5. 在 Deliverables 标签页中可以看到链接到 `test-page.html` 的条目  
6. 任务在通过评审后会实时移动到 REVIEW / DONE  
7. 以上所有过程都在**无需刷新页面**的情况下完成

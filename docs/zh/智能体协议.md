# 智能体协议（Agent Protocol）

本文档描述 OpenClaw 智能体如何与 Mission Control 进行交互。

## 任务分配流程（Task Assignment Flow）

1. **人类在 Mission Control UI 中分配任务**
   - 在看板上将任务卡片拖动到某个智能体下方的 `ASSIGNED` 列  
   - 系统会自动把任务派发到该智能体对应的 OpenClaw 会话

2. **智能体接收任务通知**

   ```text
   🔵 NEW TASK ASSIGNED

   Title: Build authentication system
   Description: Implement JWT-based auth with refresh tokens
   Priority: HIGH
   Due: 2026-02-05
   Task ID: abc-123-def

   Please work on this task. When complete, reply with:
   TASK_COMPLETE: [brief summary of what you did]

   If you need help or clarification, ask me (the orchestrator).
   ```

3. **智能体开始处理任务**
   - 任务会自动移动到 `IN PROGRESS` 列  
   - 智能体状态变为 `working`  
   - 智能体可以通过正常对话向 orchestrator（主调度智能体）发问或请求帮助

4. **智能体完成任务**
   - 智能体发送完成消息：

     ```text
     TASK_COMPLETE: Built JWT authentication system with access/refresh tokens,
     middleware for protected routes, and secure token storage.
     ```

   - 任务会自动移动到 `REVIEW` 列  
   - 智能体状态回到 `standby`

5. **orchestrator 进行审核**
   - orchestrator 查看该智能体的会话历史  
   - orchestrator 检查交付物 / 代码  
   - 如果通过：orchestrator 将任务移动到 `DONE`  
   - 如果需要修改：orchestrator 带着反馈把任务移回去

## 完成消息格式（Completion Message Format）

```text
TASK_COMPLETE: [concise summary of what you accomplished]
```

建议尽量附带“凭证”（receipts）：

```text
TASK_COMPLETE: <summary> | deliverables: <paths/links> | verification: <how you verified>
```

## 进度更新（防止任务停滞）

智能体应该定期发送进度更新，这样 orchestrator 可以尽快发现问题并解堵。

格式示例：

```text
PROGRESS_UPDATE: <what changed> | next: <next step> | eta: <time>
```

## 阻塞情况（Blockers：显式说明 + 并行兜底）

如果你被卡住了，**不要默不作声**。

格式示例：

```text
BLOCKED: <what is blocked> | need: <specific input> | meanwhile: <fallback work>
```

规则：**一边提问，一边继续做当前能做的最佳下一步**。

### 示例

✅ 好例子：

```text
TASK_COMPLETE: Refactored authentication module to use async/await,
added unit tests, and updated documentation.
```

✅ 好例子：

```text
TASK_COMPLETE: Researched 5 competitor pricing models, compiled findings
in pricing-analysis.md with recommendations.
```

❌ 不好（太模糊）：

```text
TASK_COMPLETE: Done
```

❌ 不好（缺少前缀）：

```text
I finished the task successfully!
```

## 获取帮助（Getting Help）

如果你被卡住或需要澄清需求，可以：

1. **在当前会话里直接 @ orchestrator**

   ```text
   @the orchestrator - Question about the authentication task: Should we support
   OAuth providers or just email/password for now?
   ```

2. **请求与其他智能体协作**

   ```text
   @the orchestrator - I need help from Design agent to create the login UI.
   Can you coordinate?
   ```

3. **上报阻塞**

   ```text
   @the orchestrator - Blocked on this task: Missing API credentials for the
   third-party service. Can you provide?
   ```

## 会话管理（Session Management）

### 智能体会话

- 每个智能体都有一个持久化的 OpenClaw 会话  
- 会话 ID 统一格式：`mission-control-{agent-name}`  
  - 示例：`mission-control-engineering`  
  - 示例：`mission-control-writing`

### 会话绑定

- 当第一次给智能体分配任务时，会自动把它与对应的 OpenClaw 会话绑定  
- 该会话会持续存在，用于后续任务  
- orchestrator 可以在 Mission Control UI 中手动绑定 / 解绑智能体与会话

## 状态流转（Status Transitions）

### 任务状态（Task Statuses）

- **INBOX**：未分配，等待分拣  
- **ASSIGNED**：已分配给某个智能体，系统已自动派发  
- **IN PROGRESS**：智能体正在处理  
- **REVIEW**：智能体标记完成，等待 orchestrator 审核  
- **DONE**：审核通过，任务关闭

### 智能体状态（Agent Statuses）

- **standby**：空闲，可接新任务  
- **working**：当前有正在处理的任务  
- **offline**：未连接到 OpenClaw

## API 集成方式（API Integration）

智能体**不会直接调用 Mission Control 的 HTTP API**。所有交互都通过 OpenClaw 会话完成：

1. **接收任务**：通过 OpenClaw 会话中的系统消息  
2. **汇报完成**：发送 `TASK_COMPLETE` 消息  
3. **提出问题**：通过与 orchestrator 的普通对话

Mission Control 负责：

- 任务路由  
- 状态更新  
- 事件记录  
- 工作流规则（谁能改状态、何时允许 DONE 等）

## orchestrator 的职责（The orchestrator's Responsibilities）

作为主调度智能体，orchestrator 需要：

- **分拣来自人类的任务**  
- **把任务分派给合适的专家智能体**  
- **通过会话活动监控进度**  
- **在标记 DONE 之前审查已完成的工作**  
- **在需要多智能体协作时进行协调**  
- **为被卡住的智能体提供指导**  
- **执行质量标准**

只有 orchestrator（数据库中 `is_master = 1` 的主智能体）可以把任务从 `REVIEW` 拖到 `DONE`。

## 错误处理（Error Handling）

### 如果任务派发失败

- 检查目标智能体的 OpenClaw 会话是否在线  
- 确认 Gateway 连接正常  
- 必要时可以尝试通过 API 手动派发

### 如果检测不到完成消息

- 确认消息格式是否 **严格** 为：`TASK_COMPLETE: ...`  
- 检查智能体会话是否正确绑定  
- 必要时可以在 UI 中手动变更任务状态

### 如果任务卡在 REVIEW

- orchestrator 需要手动审核并拖动到 `DONE`  
- 只有主智能体可以执行这个操作  
- 这是一个最后的质量控制关卡

## 示例工作流（Example Workflow）

```text
[Human] Creates task: "Write blog post about AI agents"
        ↓
[System] Auto-assigns to Writing agent
        ↓
[Writing] Receives notification in OpenClaw session
        ↓
[Writing] Works on blog post, saves to docs/blog/ai-agents.md
        ↓
[Writing] Replies: "TASK_COMPLETE: Wrote 1500-word blog post about
         AI agents with examples and best practices."
        ↓
[System] Auto-moves to REVIEW
        ↓
[the orchestrator] Reviews docs/blog/ai-agents.md
        ↓
[the orchestrator] Approves → moves to DONE
        ↓
[Human] Publishes blog post
```

## 最佳实践（Best Practices）

1. **在完成摘要里尽量写具体** —— 帮助 orchestrator 更快审核  
2. **尽早求助** —— 不要一个人“空转”，直接 @ orchestrator  
3. **留下痕迹** —— 把关键操作、文件路径等记录在对话或活动日志中  
4. **一次专注一个任务** —— 完成当前任务再接下一个  
5. **定期同步进度** —— 如果任务会花比较久的时间，中途要向 orchestrator 报告一下

## 未来增强（Future Enhancements）

计划中的特性：

- 进度百分比（25%、50%、75% 完成）  
- 任务依赖（任务 B 依赖任务 A）  
- 子任务拆分  
- 用时统计  
- 质量指标
